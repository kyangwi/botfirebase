# Walkthrough: Layout and Mobile Overlapping Fixes

We have modified the bot message layout order and resolved the input bar/suggestive question overlapping issues on mobile view.

## Changes Made

### 1. Chart Rendering Position
- **Updated Order:** Modified the `appendChart` rendering function in [index.html](file:///d:/JOSH/AgenticSQLChatBot/chat/templates/chat/index.html) and [templates/index.html](file:///d:/JOSH/AgenticSQLChatBot/templates/index.html) to dynamically check for Suggestive Analysis blocks (`.analysis-block`).
- **Insertion Logic:** Instead of appending the chart at the very end of the bot message (after suggestions), the chart is now inserted *before* the Suggestive Analysis blocks using jQuery's `.insertBefore()`.
- **Result:** Charts now display directly after the text generated by the LLM and before the suggested question chips, making it easy for the user to relate the chart visualizations to the explanations.

---

### 2. Question Bar Mobile Overlapping Fix
- **Responsive Flex Flow:** Added layout overrides for mobile devices (`max-width: 768px`) when the chat is in its initial centered state (`.centered-layout`).
- **Relative Flow:** Overrode the absolute positioning and transform rules on `#message-form` so it renders in the relative document flow at the bottom of the screen.
- **Scrollable Suggestions:** Set `#chat-box` to `flex: 1` with auto vertical scrolling, allowing suggested questions in the empty state to scroll underneath the input box instead of overlapping it.
